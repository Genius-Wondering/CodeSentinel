# CodeSentinel 使用说明书

> **适用对象：** 使用本系统的最终用户（无需了解技术细节）  
> **版本：** v0.3.0

---

## 一、这是什么？

**CodeSentinel** 是一个 AI 驱动的代码问答系统。

你可以把一个 Python 代码仓库（或者项目文档）"喂"给它，然后用自然语言提问：

- "JWT 认证是在哪里实现的？"
- "这个配置项 `RETRIEVER_TOP_K` 在哪里被用到？"
- "限流策略是怎么工作的？"

系统会分析代码，找到最相关的位置，并给出带文件路径和行号的答案。

---

## 二、启动系统

### 前置条件

在开始之前，你需要准备：

1. **Python 3.11+** 和 **conda**（或 pip 虚拟环境）
2. **OpenAI API Key**（用于 AI 分析，需要有效的付费账号）
3. 可选：**GNU Make**（方便一键启动）

### 第一步：配置 API Key

在项目根目录复制一份环境配置文件：

```
复制 .env.example → 命名为 .env
用文本编辑器打开 .env，填写：

OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

> ⚠️ `.env` 文件包含你的私钥，不要提交到 Git 或分享给他人。

### 第二步：安装依赖

```bash
conda create -n codesentinel python=3.11 -y
conda activate codesentinel
pip install -e . --extra-index-url https://pypi.org/simple
```

### 第三步：启动后端服务

```bash
# 方法 A（推荐，使用 Makefile）
make run

# 方法 B（手动）
uvicorn app.main:app --reload
```

看到以下输出说明启动成功：
```
INFO:     Uvicorn running on http://127.0.0.1:8000
```

### 第四步：启动前端界面

**新开一个终端窗口**，然后：

```bash
# 方法 A
make frontend

# 方法 B
streamlit run frontend/app.py
```

浏览器会自动打开，地址通常是 `http://localhost:8501`。

## 三、界面操作指南

### 3.1 主界面概览

```
┌────────────────────────────────────────────────────────────┐
│  侧边栏（左侧）          │  主区域（右侧）                   │
│                          │                                  │
│  📦 Index a Repository   │  💬 Ask a question about         │
│  ─────────────────────   │      the codebase                │
│  当前已索引: N chunks    │                                  │
│  [文本框：本地路径]      │  [输入你的问题...]               │
│  [Index Local Repo 按钮] │  [Ask 按钮]                      │
│                          │                                  │
│                          │  ─── 答案展示区域 ───            │
└────────────────────────────────────────────────────────────┘
```

### 3.2 索引代码仓库

1. 在左侧文本框中输入仓库的本地路径，例如：
   - Windows：`C:\Users\你的用户名\projects\myapp`
   - macOS/Linux：`/Users/你的用户名/projects/myapp`
2. 点击 **"Index Local Repo"** 按钮
3. 等待进度（大型仓库可能需要 1–3 分钟）
4. 出现绿色 ✅ 提示说明索引完成

> 💡 **提示：** 每次索引会替换之前的代码索引（但不会影响已上传的文档）。

### 3.3 上传项目文档（可选）

除了代码，你还可以上传 **PDF 或 Markdown 文档**（如 API 设计文档、架构说明等），系统会把代码和文档结合起来回答问题。

直接在侧边栏的 **"Document Upload"** 区域操作，无需任何工具：

1. 点击上传框或将文件拖入
2. 支持同时选择多个文件（PDF / Markdown / TXT / Word / Excel）
3. 文件选好后会显示每个文件的格式、名称和大小预览
4. 点击 **"Upload & Index All"** 开始批量上传
5. 进度条实时显示当前处理的文件，全部完成后显示汇总（共上传 N 个文件，新增 M 个 chunks）

> 💡 **提示：** 代码索引和文档索引互不干扰。上传文档不会清空代码索引；重新索引代码仓库也不会清空已上传的文档。

### 3.4 提问

1. 在主区域的文本框输入问题（支持多行，可以粘贴较长的描述）
2. 在右侧选择检索范围：
   - **All sources**（默认）：同时检索代码和文档
   - **Code only**：只检索代码
   - **Docs only**：只检索文档
3. 点击 **"Ask ›"** 按钮
3. 系统会显示：

**🧠 Agent plan（展开可查看）**  
AI 是如何理解你的问题的，以及它用了哪些关键词去搜索。

**📋 Answer**  
具体的回答，包含：
- 直接答案
- 相关代码位置（文件名 + 行号）
- 可信度评级（high / medium / low）

## 四、提问技巧

### 好的问题写法 ✅

| 问题类型 | 示例 |
|---|---|
| 定位功能 | "用户登录逻辑在哪里实现的？" |
| 理解配置 | "CHUNK_SIZE_LINES 这个参数控制什么？" |
| 追踪调用链 | "当用户调用 /ask 接口时，代码执行路径是什么？" |
| 查找 Bug | "哪里可能出现并发安全问题？" |
| 模块概览 | "app/rag 目录下各个文件分别负责什么？" |

### 不适合的问题 ❌

| 问题 | 原因 |
|---|---|
| "帮我写一段新功能代码" | 系统只负责理解和问答，不生成新代码 |
| "对比两个不同仓库的实现" | 当前只支持单次索引一个仓库 |
| "仓库外的通用知识问答" | 只能回答关于已索引内容的问题 |

## 五、常见问题

**Q: 索引很慢，正常吗？**  
A: 正常。索引需要调用 OpenAI 的 Embedding API，速度取决于仓库大小和网络状况。100 个文件约需 30–90 秒。

**Q: 报错「OPENAI_API_KEY is not configured」**  
A: 检查 `.env` 文件是否正确填写了 API Key，以及服务是否重启后加载了新配置。

**Q: 问题回答不准确怎么办？**  
A: 尝试把问题说得更具体，加入关键词（如函数名、变量名、模块名）。

**Q: 重启服务后还需要重新索引吗？**  
A: 不需要。索引数据持久化保存在 `./data/chroma` 目录，重启后自动恢复。但请注意：重启后前几次查询的速度可能稍慢，混合搜索需要首次查询后才完全激活。

**Q: 侧边栏显示「API unreachable」**  
A: 后端服务没有启动，请先运行 `make run` 或 `uvicorn app.main:app --reload`。

## 六、API 接口速查

如果需要通过程序调用（而非界面），以下是核心接口：

| 接口 | 方法 | 作用 |
|---|---|---|
| `/api/v1/health` | GET | 检查服务是否运行 |
| `/api/v1/stats` | GET | 查看已索引数量 |
| `/api/v1/index/local` | POST | 索引本地仓库 |
| `/api/v1/index/upload` | POST | **上传文件并索引**（multipart，支持 PDF/MD/TXT/DOCX/XLSX） |
| `/api/v1/index/document` | POST | 通过服务器本地路径索引文档（高级用法） |
| `/api/v1/ask` | POST | 提问 |

完整接口文档（含请求/响应示例）：启动服务后访问 `http://localhost:8000/docs`
